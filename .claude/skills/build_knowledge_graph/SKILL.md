# build_knowledge_graph - 知识图谱构建器

## 🎯 核心功能

从文档中提取知识点并构建知识图谱，支持两种模式：
1. **单文档模式**：提取知识点，构建学习路径和知识体系
2. **多文档模式**：抽取实体和关系，构建跨文档的知识关联网络

## 📋 工作流SOP

**工作流SOP**：
```
1. 接收文档集合
2. 分析构建模式（单文档/多文档）
3. 提取知识点和实体
4. 识别关系和关联
5. 构建图谱结构
6. 验证图谱完整性
7. 返回知识图谱
```

**核心步骤说明**：
- **文档接收**：验证文档完整性，解析构建模式（单文档/多文档）
- **知识点提取**：单文档模式提取核心概念、定义、流程；多文档模式识别命名实体
- **关系识别**：单文档模式建立层级依赖关系；多文档模式分析实体共现和语义关系
- **图谱构建**：创建节点和边，构建邻接矩阵，建立属性映射
- **完整性验证**：检查节点边完整性，验证逻辑一致性，检测循环引用
- **结果返回**：生成标准化数据结构，提供图谱度量指标

详细流程说明请参考[REFERENCE.md](REFERENCE.md)

## 🚀 快速开始

### 基本使用（单文档模式）
```
使用 build_knowledge_graph 从培训材料构建学习路径
文档内容：新员工入职培训手册
构建模式：单文档学习路径
```

### 基本使用（多文档模式）
```
使用 build_knowledge_graph 构建HR政策知识图谱
文档集合：年假政策、病假政策、产假政策
构建模式：多文档实体关系
```

### 支持的图谱类型
- **学习路径图谱**：知识点、概念、定义、流程
- **实体关系图谱**：人员、组织、概念等实体及关系
- **混合图谱**：结合学习路径和实体关系

## 📋 输入规范

### 必需输入
```json
{
  "documents": [
    {
      "doc_id": "doc_001",
      "content": "员工年假管理规定..."
    }
  ],
  "build_mode": "single_doc_learning_path"
}
```

### 可选输入
```json
{
  "entity_types": {
    "person": ["员工", "主管", "HR"],
    "concept": ["年假", "病假", "产假"]
  },
  "knowledge_domains": ["公司文化", "规章制度", "工作流程"]
}
```
