# normalize_knowledge_format - 知识格式规范化器

## 🎯 核心功能

分析知识内容的格式不一致性，应用统一的格式标准，统一术语和表达方式，优化内容结构，确保知识格式的一致性和规范性。

## 📋 工作流SOP

**工作流SOP**：
```
1. 接收原始知识内容
2. 分析格式不一致性
3. 应用格式标准
4. 统一术语和表达
5. 优化内容结构
6. 验证格式一致性
7. 返回规范化内容
```

### 详细流程说明

**步骤1：接收原始知识内容**
- 验证输入内容的完整性和有效性
- 检查格式标准和术语指南的有效性
- 解析结构模板要求
- 记录内容基本信息

**步骤2：分析格式不一致性**
- 识别文档结构的差异
- 分析术语使用的不一致
- 检查表达方式的差异
- 生成不一致性分析报告

**步骤3：应用格式标准**
- 应用文档结构标准
- 统一标题层次和段落格式
- 标准化列表样式
- 生成标准化结构

**步骤4：统一术语和表达**
- 应用术语指南
- 统一专业术语和常用词汇
- 标准化缩写词使用
- 生成术语统一报告

**步骤5：优化内容结构**
- 优化段落组织
- 调整信息层次
- 改善内容流畅性
- 生成优化后的结构

**步骤6：验证格式一致性**
- 检查格式的一致性
- 验证术语的统一性
- 评估结构质量
- 计算一致性评分

**步骤7：返回规范化内容**
- 生成规范化后的内容
- 提供格式变更记录
- 返回一致性评分
- 提供合规性报告

## 🚀 快速开始

### 基本使用
```
使用 normalize_knowledge_format 规范化HR政策文档格式
格式标准：企业知识管理标准v2.0
```

### 支持的格式类型
- **文档结构**: 标题层次、段落格式、列表样式
- **术语使用**: 专业术语、常用词汇、缩写词
- **表达方式**: 语言风格、句式结构、用词规范

## 📋 输入规范

### 必需输入
```json
{
  "raw_knowledge": "需要规范化的原始知识内容...",
  "format_standards": {
    "document_structure": "standard_template_v2",
    "terminology_guide": "hr_terminology_dict"
  }
}
```

## 📤 输出内容

### 标准输出
```json
{
  "normalized_content": "规范化后的知识内容...",
  "format_changes": [
    {
      "change_type": "terminology",
      "original": "年假",
      "normalized": "年度休假"
    }
  ],
  "consistency_score": 0.96
}
```

## 📊 质量指标

- **格式一致性**: ≥95%（目标值）
- **术语统一性**: ≥90%（目标值）
