# segment_knowledge_content - 知识片段切分器

## 🎯 核心功能

分析知识内容的结构，识别逻辑切分点，执行内容切分，验证切分后片段的连贯性，并建立片段之间的关联关系。

## 📋 工作流SOP

**工作流SOP**：
```
1. 接收知识内容
2. 分析内容结构
3. 识别逻辑切分点
4. 执行内容切分
5. 验证片段连贯性
6. 建立片段关联
7. 返回知识片段
```

### 详细流程说明

**步骤1：接收知识内容**
- 验证输入内容的完整性和有效性
- 检查内容长度是否符合切分要求
- 解析切分策略和片段大小参数
- 记录内容基本信息

**步骤2：分析内容结构**
- 识别文档的章节结构
- 分析段落之间的逻辑关系
- 识别主题转换和过渡点
- 生成内容结构图谱

**步骤3：识别逻辑切分点**
- 基于章节标题识别切分点
- 分析主题转换的自然断点
- 考虑内容的依赖关系
- 生成候选切分点列表

**步骤4：执行内容切分**
- 按照切分策略执行内容分割
- 确保每个片段的完整性
- 保持片段的适当大小
- 生成初步的知识片段

**步骤5：验证片段连贯性**
- 评估每个片段的内部连贯性
- 检查片段是否包含完整语义
- 验证片段的可理解性
- 计算连贯性评分

**步骤6：建立片段关联**
- 分析片段之间的逻辑关系
- 建立片段间的引用关系
- 识别前置和后置片段
- 生成片段依赖图

**步骤7：返回知识片段**
- 生成结构化的片段列表
- 包含片段ID、内容、类型、连贯性评分
- 提供切分质量评估
- 返回片段依赖关系图

## 🎯 适用场景

- **文档重构**: 将长文档切分为适合在线学习的模块
- **分批处理**: 将大文档切分为小块进行并行处理
- **内容重组**: 按不同维度重新组织文档结构
- **学习路径规划**: 将培训材料切分为学习单元

## 🚀 快速开始

### 基本使用
```
使用 segment_knowledge_content 切分HR政策文档
切分策略：按章节和主题切分
```

### 支持的切分策略
- **章节切分**: 按文档章节结构切分
- **主题切分**: 按主题和子主题切分
- **逻辑切分**: 按逻辑关系和依赖切分
