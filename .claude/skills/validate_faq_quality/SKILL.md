# validate_faq_quality - FAQ质量验证器

## 🎯 核心功能

验证FAQ集合的质量，包括答案准确性、问题覆盖度、语言质量和实用价值评估，计算综合质量评分并生成改进建议。

## 📋 工作流SOP

**工作流SOP**：
```
1. 接收FAQ集合和源文档
2. 验证答案准确性
3. 检查问题覆盖度
4. 评估语言质量
5. 测试实用价值
6. 计算综合质量评分
7. 生成改进建议
8. 返回质量报告
```

### 详细流程说明

**步骤1：接收FAQ集合和源文档**
- 验证FAQ集合的完整性
- 检查源文档的有效性
- 解析质量标准参数
- 记录验证请求基本信息

**步骤2：验证答案准确性**
- 对比FAQ答案与源文档内容
- 检查事实性错误和逻辑矛盾
- 验证数据准确性
- 计算准确性评分

**步骤3：检查问题覆盖度**
- 分析源文档的关键主题
- 检查FAQ是否覆盖主要知识点
- 识别缺失的重要问题
- 计算覆盖度评分

**步骤4：评估语言质量**
- 评估问题表述的清晰度
- 检查答案的完整性和条理性
- 测试可读性评分
- 计算语言质量评分

**步骤5：测试实用价值**
- 评估问题的实际使用频率
- 检查答案的实用性
- 验证问题-答案匹配度
- 计算实用价值评分

**步骤6：计算综合质量评分**
- 综合各维度评分
- 计算总体质量评分
- 识别主要问题领域
- 生成质量评估报告

**步骤7：生成改进建议**
- 基于问题分析生成建议
- 设定建议的优先级
- 估算改进工作量
- 生成可执行的改进方案

**步骤8：返回质量报告**
- 返回总体质量评分
- 提供各维度评分详情
- 列出发现的问题
- 提供改进建议

## 🚀 快速开始

### 基本使用
```
使用 validate_faq_quality 验证年假政策FAQ质量
FAQ集合：年假政策FAQ列表
源文档：年假政策完整文档
```

### 支持的验证维度
- **准确性**: 答案是否准确，是否符合源文档
- **覆盖度**: 问题是否全面覆盖源文档内容
- **清晰度**: 语言表达是否清晰易懂
- **实用性**: 是否具有实际使用价值

## 📋 输入规范

### 必需输入
```json
{
  "faq_collection": "待验证的FAQ集合",
  "source_documents": "相关的源文档",
  "quality_standards": {
    "accuracy_threshold": 0.9,
    "coverage_requirement": 0.85,
    "readability_standard": 4.0
  }
}
```

```json
{
  "validation_depth": "验证深度（basic/detailed/comprehensive）",
  "focus_areas": "重点关注领域"
}
```
