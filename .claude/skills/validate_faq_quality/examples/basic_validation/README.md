- `faq_sample.md` - 待验证的FAQ集合

## 🚀 快速开始

### 1. 准备文件
确保你有以下文件：
- 源文档（政策、手册等）
- FAQ集合（Markdown格式）

### 2. 运行验证

## 🎯 验证维度说明

### 1. 答案准确性 (Accuracy)
- 检查FAQ答案是否与源文档内容一致
- 识别事实性错误和逻辑矛盾
- 权重: 30%

### 2. 问题覆盖度 (Coverage)
- 分析源文档的关键主题
- 检查FAQ是否覆盖主要知识点
- 识别缺失的重要问题
- 权重: 30%

### 3. 语言清晰度 (Clarity)
- 评估问题表述的清晰度
- 检查答案的完整性和条理性
- 测试可读性评分
- 权重: 20%

### 4. 实用价值 (Usability)
- 评估问题的实际使用频率
- 检查答案的实用性
- 验证问题-答案匹配度
- 权重: 20%

## 🐛 常见问题

### Q: 验证结果显示准确性低怎么办？
A: 检查FAQ答案是否准确反映了源文档内容，确保没有遗漏关键信息或产生误解。

### Q: 覆盖度不足如何处理？
A: 分析源文档的主要章节和主题，补充相应的FAQ问题。

### Q: 清晰度评分如何提高？
A: 使用简洁明了的语言，避免长句和复杂词汇，适当使用列表和结构化格式。
