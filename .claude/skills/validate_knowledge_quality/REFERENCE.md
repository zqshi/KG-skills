# validate_knowledge_quality - 详细参考文档

## 📋 工作流SOP详细说明

### 标准执行流程
```
1. 接收验证请求 → 2. 检查内容完整性 → 3. 验证信息准确性 → 4. 评估时效性 → 5. 检查一致性 → 6. 计算质量评分 → 7. 生成改进建议 → 8. 返回验证报告
```

### 详细流程说明

#### 步骤1：接收验证请求
- 验证知识ID的有效性
- 检查质量标准配置的合理性
- 解析验证维度和权重配置
- 记录验证请求基本信息

#### 步骤2：检查内容完整性
- 验证必需字段是否存在（标题、内容、分类等）
- 检查内容覆盖度，识别缺失的关键信息
- 评估内容结构的完整性
- 生成完整性检查报告

#### 步骤3：验证信息准确性
- 对比权威来源和参考文档
- 检查事实错误和逻辑矛盾
- 验证数据的一致性和合理性
- 识别可疑或不准确的信息

#### 步骤4：评估时效性
- 检查知识的发布时间
- 识别可能过时的内容
- 评估内容的更新频率
- 标记需要更新的知识点

#### 步骤5：检查一致性
- 验证与其他知识条目的一致性
- 检查逻辑连贯性
- 识别冲突或矛盾的信息
- 生成一致性检查报告

#### 步骤6：计算质量评分
- 综合各维度评分（完整性、准确性、时效性、一致性）
- 应用权重配置计算总体质量评分
- 生成评分明细和趋势分析
- 识别质量瓶颈

#### 步骤7：生成改进建议
- 基于问题分析生成优先级建议
- 估算改进工作量和预期效果
- 提供具体的改进措施
- 设定改进优先级

#### 步骤8：返回验证报告
- 返回总体质量评分
- 提供各维度详细评分
- 列出发现的问题和严重程度
- 提供改进建议和优先级

## 🔍 验证维度详解

### 完整性验证
**检查内容**：
- 必需字段：标题、内容、分类、标签、来源
- 内容覆盖：关键信息是否完整
- 结构完整：章节结构是否合理
- 引用完整：外部引用和参考是否齐全

**评分标准**：
- 90-100分：内容完整，无缺失信息
- 80-89分：基本完整，少量非关键信息缺失
- 70-79分：部分关键信息缺失
- 60-69分：较多关键信息缺失
- 60分以下：内容严重不完整

### 准确性验证
**检查内容**：
- 事实准确性：数据、日期、名称等是否正确
- 逻辑一致性：内容逻辑是否自洽
- 来源可靠性：信息来源是否权威
- 引用正确性：引用和参考是否准确

**验证方法**：
- 与权威来源对比
- 逻辑推理验证
- 交叉验证
- 专家审核

### 时效性评估
**检查内容**：
- 发布时间：知识的创建和更新时间
- 有效期：知识的有效期限
- 更新频率：内容的更新周期
- 过时风险：内容过时的可能性

**评估标准**：
- 90-100分：内容最新，无过时风险
- 80-89分：内容较新，轻微过时风险
- 70-79分：内容一般，有一定过时风险
- 60-69分：内容较旧，过时风险较高
- 60分以下：内容过时，需要立即更新

### 一致性检查
**检查内容**：
- 内部一致性：内容内部逻辑是否一致
- 外部一致性：与其他知识条目是否一致
- 术语一致性：术语使用是否统一
- 格式一致性：格式和结构是否统一

## 📊 质量评分体系

### 评分权重配置
```yaml
quality_scoring:
  weights:
    completeness: 0.30   # 完整性权重30%
    accuracy: 0.35       # 准确性权重35%
    timeliness: 0.20     # 时效性权重20%
    consistency: 0.15    # 一致性权重15%
```

### 评分等级
- **优秀 (90-100分)**：知识质量优秀，无需改进
- **良好 (80-89分)**：知识质量良好，轻微改进
- **一般 (70-79分)**：知识质量一般，需要改进
- **较差 (60-69分)**：知识质量较差，急需改进
- **很差 (60分以下)**：知识质量很差，建议重写

## ⚠️ 注意事项详解

### 数据质量要求
- **来源可靠性**：确保验证依据的来源可靠权威
- **数据时效性**：使用最新的参考数据进行验证
- **覆盖全面性**：验证样本要具有代表性
- **标准一致性**：保持验证标准的一致性

### 验证深度要求
- **多维度验证**：从多个维度全面验证
- **深度分析**：不仅发现问题，还要分析根本原因
- **对比验证**：与历史数据和同类知识对比
- **趋势分析**：分析质量变化趋势

### 建议质量要求
- **可行性**：建议要具有可操作性
- **优先级合理**：基于影响程度和紧急性设置优先级
- **效果可量化**：预估改进效果，设定可量化的目标
- **成本效益**：考虑改进成本与收益的平衡

## 📊 质量验证指标

### 核心指标
- **验证准确率**: ≥92%（目标值）
- **误报率**: ≤5%（目标值）
- **漏报率**: ≤8%（目标值）
- **处理效率**: ≤5秒/条目（目标值）

### 评估维度
- **准确性**：验证结果的准确性
- **全面性**：问题发现的全面程度
- **实用性**：改进建议的实用价值
- **及时性**：验证的及时性

## 🔍 与相关Skill的区别详解

### validate_knowledge_quality vs validate_faq_quality
- **validate_knowledge_quality**：专注于知识库条目的全面质量验证（完整性、准确性、时效性、一致性），适用于知识库维护、入库审核、更新评估
- **validate_faq_quality**：专注于FAQ集合的专项质量验证（答案准确性、问题覆盖度、语言质量、实用价值），适用于FAQ质量评估和优化

### validate_knowledge_quality vs validate_summary_quality
- **validate_knowledge_quality**：验证完整知识条目的质量
- **validate_summary_quality**：专注于知识摘要的专项质量验证（信息覆盖度、与原文一致性、信息密度、可读性），适用于摘要生成质量评估
