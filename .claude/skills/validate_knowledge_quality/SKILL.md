# validate_knowledge_quality - 知识质量验证器

## 🎯 核心功能

全面检查知识内容的完整性、准确性、时效性和一致性，计算综合质量评分，并提供详细的改进建议。

## 📋 工作流SOP

**工作流SOP**：
```
1. 接收验证请求
2. 检查内容完整性
3. 验证信息准确性
4. 评估时效性
5. 检查一致性
6. 计算质量评分
7. 生成改进建议
8. 返回验证报告
```

### 详细流程说明

**步骤1：接收验证请求**
- 验证知识ID或内容的有效性
- 检查质量标准的合理性
- 解析验证深度要求
- 记录验证请求基本信息

**步骤2：检查内容完整性**
- 验证必需字段是否存在
- 检查内容的覆盖度
- 识别缺失的关键信息
- 计算完整性评分

**步骤3：验证信息准确性**
- 对比知识内容与权威来源
- 检查事实性错误和逻辑矛盾
- 验证数据和规则的准确性
- 计算准确性评分

**步骤4：评估时效性**
- 检查知识的发布和更新时间
- 评估内容的时效性要求
- 识别过时或需要更新的内容
- 计算时效性评分

**步骤5：检查一致性**
- 验证与其他知识的一致性
- 检查内部逻辑的连贯性
- 验证引用关系的有效性
- 计算一致性评分

**步骤6：计算质量评分**
- 综合各维度评分
- 计算总体质量评分
- 识别主要问题领域
- 生成质量评估报告

**步骤7：生成改进建议**
- 基于问题分析生成建议
- 设定建议的优先级
- 估算改进工作量
- 生成可执行的改进方案

**步骤8：返回验证报告**
- 返回总体质量评分
- 提供各维度评分详情
- 列出发现的问题
- 提供改进建议

## 🎯 适用场景

- **知识库维护**: 定期检查知识条目的准确性和时效性
- **知识入库审核**: 新知识点加入知识库前的质量检查
- **知识更新评估**: 评估知识更新后的质量变化
- **知识结构化检查**: 验证知识分类和标签体系的合理性

## 📊 质量指标

- **知识准确率**: ≥92%（目标值）
- **结构完整性**: ≥90%（目标值）
- **时效性评分**: ≥85%（目标值）
- **标签质量**: ≥88%（目标值）

## 🚀 快速开始

### 基本使用
```
使用 validate_knowledge_quality 验证年假政策知识质量
验证维度：完整性、准确性、时效性、一致性
```

### 支持的验证维度
- **完整性**: 内容是否完整，关键信息是否缺失
- **准确性**: 信息是否准确，是否符合实际情况
- **时效性**: 内容是否过时，是否需要更新
- **一致性**: 与其他知识是否一致

## 🔍 与相关Skill的区别

### 与validate_faq_quality的区别

**validate_knowledge_quality**专注于：
- 知识库条目的全面质量验证
- 验证维度：完整性、准确性、时效性、一致性
- 适用于知识库维护、入库审核、更新评估
- 关注知识结构和标签体系的合理性

**validate_faq_quality**专注于：
- FAQ集合的专项质量验证
- 验证维度：答案准确性、问题覆盖度、语言质量、实用价值
- 适用于FAQ质量评估和优化
- 关注问答对的匹配度和实用性

### 与validate_summary_quality的区别

**validate_knowledge_quality**专注于：
- 完整知识条目的多维度验证
- 包括时效性、一致性等知识库特有维度
- 验证知识的结构化质量
- 适用于知识库条目的全面审核

**validate_summary_quality**专注于：
- 知识摘要的专项质量验证
- 验证维度：信息覆盖度、与原文一致性、信息密度、可读性
- 关注摘要的信息保留和表达简洁性
- 适用于摘要生成质量评估

**使用场景选择**：
- 当验证完整知识条目时使用validate_knowledge_quality
- 当验证FAQ集合时使用validate_faq_quality
- 当验证知识摘要时使用validate_summary_quality
