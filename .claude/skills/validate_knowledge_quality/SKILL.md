# validate_knowledge_quality - 知识质量验证器

## 🎯 核心功能

全面检查知识内容的完整性、准确性、时效性和一致性，计算综合质量评分，并提供详细的改进建议。

## 📋 工作流SOP

**工作流SOP**：
```
1. 接收验证请求
2. 检查内容完整性
3. 验证信息准确性
4. 评估时效性
5. 检查一致性
6. 计算质量评分
7. 生成改进建议
8. 返回验证报告
```

**核心步骤说明**：
- **请求接收**：验证知识ID有效性，检查质量标准合理性
- **完整性检查**：验证必需字段，检查内容覆盖度，识别缺失信息
- **准确性验证**：对比权威来源，检查事实错误和逻辑矛盾
- **时效性评估**：检查发布时间，识别过时内容
- **一致性检查**：验证与其他知识的一致性，检查逻辑连贯性
- **质量评分**：综合各维度评分，计算总体质量评分
- **建议生成**：基于问题分析生成优先级建议，估算工作量
- **报告返回**：返回总体评分、各维度详情、问题列表和改进建议

详细流程说明请参考[REFERENCE.md](REFERENCE.md)

## 🎯 适用场景

- **知识库维护**: 定期检查知识条目的准确性和时效性
- **知识入库审核**: 新知识点加入知识库前的质量检查
- **知识更新评估**: 评估知识更新后的质量变化
- **知识结构化检查**: 验证知识分类和标签体系的合理性

## 📊 质量指标

- **知识准确率**: ≥92%（目标值）
- **结构完整性**: ≥90%（目标值）
- **时效性评分**: ≥85%（目标值）
- **标签质量**: ≥88%（目标值）

## 🚀 快速开始

### 基本使用
```
使用 validate_knowledge_quality 验证年假政策知识质量
验证维度：完整性、准确性、时效性、一致性
```

### 支持的验证维度
- **完整性**: 内容是否完整，关键信息是否缺失
- **准确性**: 信息是否准确，是否符合实际情况
- **时效性**: 内容是否过时，是否需要更新
- **一致性**: 与其他知识是否一致

## 🔍 与相关Skill的区别

**validate_knowledge_quality**专注于知识库条目的全面质量验证（完整性、准确性、时效性、一致性），适用于知识库维护、入库审核、更新评估。

**validate_faq_quality**专注于FAQ集合的专项质量验证（答案准确性、问题覆盖度、语言质量、实用价值），适用于FAQ质量评估和优化。

**validate_summary_quality**专注于知识摘要的专项质量验证（信息覆盖度、与原文一致性、信息密度、可读性），适用于摘要生成质量评估。
