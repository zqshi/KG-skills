# validate_summary_quality - 摘要质量验证器

## 🎯 核心功能

验证知识摘要的质量，包括关键信息覆盖度、与原文一致性、信息密度和可读性评估，计算综合质量评分并生成改进建议。

## 📋 工作流SOP

**工作流SOP**：
```
1. 接收摘要和源文档
2. 检查关键信息覆盖度
3. 验证摘要与原文一致性
4. 评估信息密度和简洁性
5. 测试可读性和理解度
6. 计算综合质量评分
7. 生成改进建议
8. 返回质量报告
```

### 详细流程说明

**步骤1：接收摘要和源文档**
- 验证摘要内容的完整性
- 检查源文档的有效性
- 解析质量标准参数
- 记录验证请求基本信息

**步骤2：检查关键信息覆盖度**
- 提取源文档的关键信息点
- 检查摘要是否包含所有关键点
- 识别缺失的重要信息
- 计算覆盖度百分比

**步骤3：验证摘要与原文一致性**
- 对比摘要内容与原文内容
- 检查事实性错误和逻辑矛盾
- 验证数据准确性
- 计算一致性评分

**步骤4：评估信息密度和简洁性**
- 分析摘要的信息密度
- 评估表达的简洁性
- 识别冗余内容
- 计算简洁性评分

**步骤5：测试可读性和理解度**
- 评估语言流畅度
- 测试理解难度
- 计算可读性评分
- 评估专业术语使用

**步骤6：计算综合质量评分**
- 综合各维度评分
- 计算总体质量评分
- 识别主要问题领域
- 生成质量评估报告

**步骤7：生成改进建议**
- 基于问题分析生成建议
- 设定建议的优先级
- 估算改进工作量
- 生成可执行的改进方案

**步骤8：返回质量报告**
- 返回总体质量评分
- 提供各维度评分详情
- 列出发现的问题
- 提供改进建议

## 🚀 快速开始

### 基本使用
```
使用 validate_summary_quality 验证年假政策摘要质量
摘要内容：年假政策摘要
源文档：年假政策完整文档
```

### 支持的验证维度
- **覆盖度**: 关键信息是否完整覆盖
- **准确性**: 摘要与原文是否一致
- **简洁性**: 信息密度和表达简洁度
- **可读性**: 语言流畅度和理解难度

## 📋 输入规范

### 必需输入
```json
{
  "summary_content": "待验证的摘要",
  "source_document": "源文档",
  "quality_standards": {
    "coverage_threshold": 0.85,
    "consistency_requirement": 0.9,
    "conciseness_standard": 4.0
  }
}
```

```json
{
  "validation_depth": "验证深度（basic/detailed/comprehensive）",
  "key_points_definition": "关键点定义"
}
```
